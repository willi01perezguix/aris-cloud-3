# ARIS_CONTROL_2 — Cadencia Operativa Semanal (Handoff) v1

## Estado
- **Estado actual:** `ACTIVO COMO MARCO / EJECUCIÓN BLOQUEADA HASTA GATE GO`

## Cadencia propuesta
- **Lunes (Kickoff operativo):** priorización de riesgos, revisión Bug Bar, asignación de owners.
- **Martes-Miércoles (Estabilización):** ejecución de pruebas críticas y correcciones SEV2/SEV3.
- **Jueves (Readiness):** verificación de criterios GO/NO-GO para ventana de release.
- **Viernes (Cierre/handoff):** resumen semanal, deuda abierta, plan siguiente semana.

## Rituales mínimos
1. Daily ops de 15 min con tablero de incidentes.
2. Triage de defectos 3 veces por semana.
3. Revisión de indicadores de estabilidad y error budget semanal.
4. Handoff formal semanal con minuta y responsables.

## Entregables semanales
- Estado GO/NO-GO.
- Riesgos abiertos con ETA.
- Incidentes cerrados/abiertos por severidad.
- Cambios aprobados para siguiente ciclo.
