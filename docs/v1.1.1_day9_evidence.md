# v1.1.1 Day 9 — Evidencia técnica

## Resumen de cambios
1. **Telemetría de polling de exports (P0 #1)**
   - Se añadió `telemetry_hook` opcional a `wait_for_export_ready`.
   - El hook recibe eventos por estado con `attempt`, `elapsed_ms` y `is_terminal`.
2. **Jitter configurable de retries (P0 #2)**
   - Nuevas variables de config: `ARIS3_RETRY_JITTER_ENABLED`, `ARIS3_RETRY_JITTER_MIN_SECONDS`, `ARIS3_RETRY_JITTER_MAX_SECONDS`.
   - `HttpClient` calcula delay de retry con backoff + jitter (solo si está habilitado).
3. **CLI smoke SDK (P0 #3)**
   - `post_release_smoke.py` ahora acepta `--artifacts-dir` y `--suite-name`.
   - Se mantiene salida con `exit code` 0/1 y artifacts JSON/log.

## Riesgos residuales
- El hook de telemetría es in-process; para observabilidad distribuida aún falta integración directa con backend/collector externo.
- El jitter configurado por entorno requiere alineación operativa para no divergir entre entornos.
- CLI smoke valida contrato esencial, no sustituye smoke end-to-end contra ambiente real.
