# v1.1.1 Day 9 Scope — Top 3 P0

## Criterio aplicado
Se seleccionaron exactamente 3 ítems P0 del backlog v1.1.1 usando:
1. Mayor impacto de usuario.
2. Menor riesgo de regresión.
3. Menor acoplamiento con módulos fuera del SDK.

## Ítem P0 #1 — Telemetría de polling por estado
- **Backlog:** #1 `Telemetría de polling por estado (CREATED, READY, FAILED)`.
- **Motivo:** mejora observabilidad operacional del flujo de exports (alta visibilidad y diagnóstico rápido).
- **Riesgo:** bajo; se implementa como parámetro opcional (`telemetry_hook`) sin romper firmas existentes.
- **Criterio de aceptación Day 9:**
  - `wait_for_export_ready` emite eventos por estado observado.
  - Evento terminal incluye latencia (`elapsed_ms`) y marca terminal.
  - Cobertura unitaria validando secuencia `CREATED -> READY`.

## Ítem P0 #2 — Retries con jitter configurable
- **Backlog:** #2 `Retries con jitter configurable para requests idempotentes`.
- **Motivo:** reduce reintentos sincronizados y picos de carga en fallos transitorios.
- **Riesgo:** bajo-medio; comportamiento default se mantiene (jitter desactivado por defecto).
- **Criterio de aceptación Day 9:**
  - Config expone `ARIS3_RETRY_JITTER_ENABLED`, `ARIS3_RETRY_JITTER_MIN_SECONDS`, `ARIS3_RETRY_JITTER_MAX_SECONDS`.
  - Delay de retry queda dentro de límites esperados cuando jitter está activo.
  - Validaciones de configuración cubren tipos y rangos.

## Ítem P0 #3 — CLI utilitaria para smoke SDK
- **Backlog:** #3 `CLI utilitaria para smoke SDK (config + exports + cache)`.
- **Motivo:** soporte/operaciones puede ejecutar validación rápida post-release con evidencia JSON/log.
- **Riesgo:** bajo; extensión compatible sobre script existente.
- **Criterio de aceptación Day 9:**
  - CLI permite parámetros de salida (`--artifacts-dir`) y suite (`--suite-name`).
  - Mantiene checks de config strict + exports polling + cache.
  - Retorna exit code `0/1` y genera artifact JSON + log.
